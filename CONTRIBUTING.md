# Contributing to martsmart.github.io

Thank you for your interest in contributing to this trading strategy project!

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your changes
4. Make your modifications
5. Commit with clear messages
6. Push to your fork
7. Open a Pull Request

## Code Guidelines

- Write clear, descriptive commit messages
- Add comments to complex trading logic
- Test strategies with historical data first
- Document any new indicators or parameters
- Follow existing code style

## Pull Request Process

1. Update the README.md if needed
2. Include a clear description of changes
3. Link any related issues
4. Ensure your code doesn't break existing functionality
5. Wait for review and approval

## Testing

- Test all strategies with backtesting first
- Verify paper trading execution
- Document test results
- Check for edge cases

## Reporting Issues

- Provide clear, detailed descriptions
- Include error messages and logs
- Specify your environment (Python version, OS, etc.)
- Include steps to reproduce

## Questions?

Feel free to open an issue or discussion for questions about the project.

## Code of Conduct

- Be respectful and professional
- No harassment or discrimination
- Provide constructive feedback
- Collaborate in good faith
